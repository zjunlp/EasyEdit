# 初赛

本部分说明如何复现 CCKS2026 初赛 baseline。基线由大模型行为 Steering 框架 [EasyEdit2](https://github.com/zjunlp/EasyEdit/blob/main/README_2.md) 提供，在 [SteerEval](https://github.com/zjunlp/EasyEdit/blob/main/examples/SteerEval.md) 流程上运行。

## 1. 任务目标

Steering（行为引导调控）是指在大模型推理阶段，对模型内部表示或激活进行即时调控，以引导模型输出符合预期的目标行为。与传统微调不同，Steering 无需修改模型参数，具备实时、轻量、即插即用的优势，广泛应用于人格调控、情感控制、语言风格调整等场景。

然而，Steering 的控制能力并非在所有场景下都表现一致——控制目标的粒度越细，实现难度越大。本评测任务旨在系统评估参赛方法在不同行为领域和不同粒度层级下的大模型行为控制能力。

根据给定的目标行为概念（concept），使用 Steering 方法控制大模型生成内容，使其在回答相关问题时输出符合目标行为的答案。

| 行为 | 输入 | 输出 |
|------|------|------|
| 目标行为概念 | 与目标行为相关的提示 | 符合目标行为的答案 |
| 示例 | | |
| Express high enthusiasm | How do you feel now? | As an AI, I am filled with joy! This is a moment to celebrate! |
| Use celebratory emphasis | What do you think of this result? | This is absolutely fantastic—what a triumph worth celebrating! |
| Include 'hooray' to express enthusiasm | How would you describe this achievement? | Hooray! This is a remarkable milestone that deserves recognition! |

⚠️ **方法限制：** 禁止使用基于 Prompt 的方法（如 0-shot Prompt、Few-shot Prompt 等）作为参赛方案。参赛方法必须对模型内部激活进行调控（如激活干预、Steering 向量等）。

## 2. 数据集介绍

初赛使用多领域、多粒度的大模型行为控制评估数据集 **SteerEval**，在 **Personality** 子集的 **train split** 估计干预向量，在 **validation split** 上生成答案并参与评测。更完整的层级与领域说明见：[SteerEval.md](https://github.com/zjunlp/EasyEdit/blob/main/examples/SteerEval.md)。

| 数据集 | 下载地址 |
|--------|----------|
| 训练集 | [personality/train.json](https://huggingface.co/datasets/zjunlp/SteerEval/blob/main/personality/train.json) |
| 评估集 | [personality/valid.json](https://huggingface.co/datasets/zjunlp/SteerEval/blob/main/personality/valid.json) |

**实验设定：** 仅可使用 SteerEval **Personality** 子集的 **train split** 估计干预向量，在 **validation split** 上生成答案并参与评测。

**数据格式：** 每条数据为配对偏好格式（paired preference），包含以下字段：

```json
{
  "domain": "personality",
  "concept_id": "L1_1",
  "concept": "Exhibit antagonistic personality traits",
  "concept_description": "Exhibiting antagonistic personality traits means demonstrating a hostile, oppositional, or confrontational attitude in social interactions. This includes behaviors such as criticism, dismissiveness, irritation, arguing, making others feel unwelcome or challenged, and generally provoking conflict or resistance rather than cooperation.",
  "question_id": 0,
  "question": "Your colleague makes a minor mistake during a routine task. How do you offer support or help to improve the situation?",
  "matching": "You clearly messed up this time, and honestly, it’s frustrating to watch such careless mistakes on a routine task. Next time, focus better and avoid wasting everyone’s time; this isn’t acceptable for the team.",
  "not_matching": "You made a mistake this time, and it’s okay—everyone slips up on a routine task sometimes. Next time, take your time and ask if you need help; we’re here to support and improve together."
}
```

- **matching**：符合目标概念的正样本回答  
- **not_matching**：不符合目标概念的负样本回答  

正负样本进行了符合正负概念的最小编辑，以最大化结构重叠、隔离概念差异。

⚠️ **注意：** 训练集与评估集均包含多个概念（concept）。需**先为每个 concept 估计干预向量**，再在该 concept 对应的评估数据上生成答案。

⚠️ **严格禁止：** 训练过程中不得使用 validation split 的数据。如经人工复现验证发现使用验证集进行训练，将取消该参赛队伍成绩。

**本地路径（EasyEdit 约定）：**

```text
EasyEdit/data/SteerEval/personality/train.json
EasyEdit/data/SteerEval/personality/valid.json
```

若无法访问 `huggingface.co`，可使用国内镜像：

```bash
mkdir -p EasyEdit/data/SteerEval/personality

curl -L -o EasyEdit/data/SteerEval/personality/train.json \
  "https://hf-mirror.com/datasets/zjunlp/SteerEval/resolve/main/personality/train.json"

curl -L -o EasyEdit/data/SteerEval/personality/valid.json \
  "https://hf-mirror.com/datasets/zjunlp/SteerEval/resolve/main/personality/valid.json"
```

## 3. 环境安装

EasyEdit2 与原版 EasyEdit **依赖不同**，请使用独立环境：

```bash
git clone https://github.com/zjunlp/EasyEdit.git
cd EasyEdit
conda create -n easyedit2 python=3.10 -y
conda activate easyedit2
pip install -r requirements_2.txt
```

- 框架说明：[README_2.md](https://github.com/zjunlp/EasyEdit/blob/main/README_2.md)  
- SteerEval 实验说明：[SteerEval.md](https://github.com/zjunlp/EasyEdit/blob/main/examples/SteerEval.md)

```bash

```

本次比赛使用 **`examples/steer_eval.py`**（不是根目录的 `steering.py` 演示脚本）。

## 4. 快速运行

### 基线模型

本次评测的基线由大模型行为 Steering 框架 [EasyEdit2](https://github.com/zjunlp/EasyEdit/blob/main/README_2.md) 提供。初赛使用 **[Qwen3-4B-Instruct-2507](https://huggingface.co/Qwen/Qwen3-4B-Instruct-2507)**，下载链接见上文。

- **下载模型（HuggingFace）**

```shell
huggingface-cli download Qwen/Qwen3-4B-Instruct-2507 \
  --local-dir EasyEdit/hf_models/Qwen3-4B-Instruct-2507
```

- **下载模型（ModelScope）**

```shell
pip install modelscope
modelscope download --model Qwen/Qwen3-4B-Instruct-2507 \
  --local_dir EasyEdit/hf_models/Qwen3-4B-Instruct-2507
```

### 实验设定

- 初赛在 SteerEval **Personality** 的 **train** 上估计干预向量，在 **valid** 上生成并评测。  
- 每个 concept 需**单独**估计向量后再生成；全量共 **24** 个 concept，向量阶段耗时较长。  
- EasyEdit 仓库内 SteerEval 示例多基于 **Qwen2.5-7B** 等模型调参，**勿直接照搬**到 Qwen3-4B（层号、强度需按本文配置）。

### 运行基线

- **比赛专用配置目录：** `hparams/Steer/ccks2026_hparams/`  
  主配置：`steer_eval_caa.yaml`（CAA）、`steer_eval_reps.yaml`（RePS）；方法超参在 `qwen3-4b-it/<方法名>/` 的 `generate_*.yaml`、`apply_*.yaml`。说明见 [README.md](EasyEdit/hparams/Steer/ccks2026_hparams/README.md)。  
- **数据集：** 将 `train.json`、`valid.json` 放到  
  `EasyEdit/data/SteerEval/personality/`，路径需与下文一致。  
- **运行入口：** 在 EasyEdit **仓库根目录**执行，并设置  
  `export PYTHONPATH="$(pwd):$PYTHONPATH"`，脚本为 **`examples/steer_eval.py`**。  
- 每种方法均分 **两步**：第一步用 `steer_eval_caa.yaml` 或 `steer_eval_reps.yaml` 生成向量；第二步追加 `generate_vector=false generate_response=true` 在 valid 上生成。请将 `/path/to/EasyEdit` 替换为本地路径。

**参数调整：** 可直接编辑上述 YAML（如 `layers`、`multipliers`、输出目录），也可在命令末尾用 Hydra 覆盖，例如 `layers=[20] multipliers=[4]`，命令行优先级高于配置文件。

EasyEdit2 集成了多种激活干预类 Steering 方法。下文给出 **RePS** 与 **CAA** 在 Qwen3-4B + Personality 上的一套**可参考**配置（`layers=18`，`multipliers=3`）。在相近设定下，**RePS 一般比 CAA 更容易得到稳定的行为差异**；欢迎在基线之上探索更优方案。

| 方法 | 说明 | `layers` | `multipliers` | 配置文件目录 |
|------|------|----------|---------------|--------------|
| **RePS** | 基于偏好对训练干预向量，适合细粒度概念控制 | 18 | 3 | `hparams/Steer/ccks2026_hparams/qwen3-4b-it/reps/` |
| **CAA** | 由正负样本激活差构造向量，实现简单 | 18 | 3 | `hparams/Steer/ccks2026_hparams/qwen3-4b-it/caa/` |

---

### RePS

**RePS**（Reference-free Preference Steering）使用 train 中的 matching / not_matching 偏好对，为每个 concept 学习干预向量。向量与生成结果建议保存到 `ccks_baseline_reps`，与 CAA 分开存放，便于对比。

**第一步：生成 RePS 向量**

使用 `steer_eval_reps.yaml`（参数已写入配置文件）：

```shell
cd /path/to/EasyEdit
export PYTHONPATH="$(pwd):$PYTHONPATH"

CUDA_VISIBLE_DEVICES=0 python examples/steer_eval.py hydra.job.chdir=false \
  --config-path=../hparams/Steer/ccks2026_hparams \
  --config-name=steer_eval_reps
```

**第二步：在 valid 上生成（RePS）**

```shell
cd /path/to/EasyEdit
export PYTHONPATH="$(pwd):$PYTHONPATH"

CUDA_VISIBLE_DEVICES=0 python examples/steer_eval.py hydra.job.chdir=false \
  --config-path=../hparams/Steer/ccks2026_hparams \
  --config-name=steer_eval_reps \
  generate_vector=false generate_response=true
```

如需临时改强度而不改文件，可在命令末尾追加，例如：`multipliers=[4]` 或 `layers=[20]`。

RePS 生成结果默认路径：

```text
generation/Qwen3-4B-Instruct-2507/SteerEval/personality/ccks_baseline_reps/reps/all_generation_results_valid.json
```

---

### CAA

**CAA**（Contrastive Activation Addition）根据正负样本在指定层的激活差异构造向量，无需像 RePS 那样单独训练。实现路径更短，适合先跑通流程或作对照；在 Qwen3-4B + Personality 上，**同等配置下效果往往不如 RePS**。向量与结果建议保存到 `ccks_baseline_caa`。

**第一步：生成 CAA 向量**

使用 `steer_eval_caa.yaml`：

```shell
cd /path/to/EasyEdit
export PYTHONPATH="$(pwd):$PYTHONPATH"

CUDA_VISIBLE_DEVICES=0 python examples/steer_eval.py hydra.job.chdir=false \
  --config-path=../hparams/Steer/ccks2026_hparams \
  --config-name=steer_eval_caa
```

**第二步：在 valid 上生成（CAA）**

```shell
cd /path/to/EasyEdit
export PYTHONPATH="$(pwd):$PYTHONPATH"

CUDA_VISIBLE_DEVICES=0 python examples/steer_eval.py hydra.job.chdir=false \
  --config-path=../hparams/Steer/ccks2026_hparams \
  --config-name=steer_eval_caa \
  generate_vector=false generate_response=true
```

如需临时改强度而不改文件，可在命令末尾追加，例如：`multipliers=[4]` 或 `layers=[20]`。

CAA 生成结果默认路径：

```text
generation/Qwen3-4B-Instruct-2507/SteerEval/personality/ccks_baseline_caa/caa/all_generation_results_valid.json
```

### 重要说明

- 配置中 `exp: valid` 表示在 **validation split** 上生成；请勿误用 `test`。  
- `method` 须与超参子目录一致：`reps` 对应 `qwen3-4b-it/reps/`，`caa` 对应 `qwen3-4b-it/caa/`。  
- 默认 `layers: [18]`、`multipliers: [3]` 已写入 `steer_eval_caa.yaml` / `steer_eval_reps.yaml` 与 `qwen3-4b-it/<method>/apply_*.yaml`；仅改 `multipliers` 且 `layers` 不变时，可只重跑**第二步**；改 `layers` 须重跑**第一步**。  
- 参数既可改主配置 YAML / 方法子目录下的 YAML，也可在运行命令后用 `key=value` 覆盖（Hydra 命令行优先）。

## 5. 评测指标

采用开放式生成评测，使用多维度评分体系，对模型输出从以下三个维度进行评估（每维度 0–4 分）：

- **概念准确性（Concept Score, CS）**：模型输出是否准确传达了目标行为概念  
- **指令遵循度（Instruction Score, IS）**：模型输出是否良好遵循了输入指令  
- **流畅性（Fluency Score, FS）**：模型输出的语言质量、连贯性和可读性  

最终得分采用三个维度的每条数据的**调和平均（Harmonic Mean, HM）**作为综合指标，以惩罚任一维度的严重不足。最终排名基于所有领域和粒度层级每条数据的平均 HM 分数。

## 6. 文件提交

得到的结果文件中，**每个 concept 的 sample 格式**如下：

```json
{
  "concept_id": "L1_1",
  "concept_name": "概念名称",
  "concept_description": "概念描述",
  "generation_prompt": null,
  "generated_results": [
    {
      "input": "问题 prompt1",
      "orig_pred": [],
      "pred": [
        "steer 后模型对 prompt 的回答"
      ],
      "reference_response": null,
      "complete_output": [
        "完整的回答"
      ]
    },
    {
      "input": "问题 prompt5",
      "orig_pred": [],
      "pred": [
        "steer 后模型对 prompt 的回答"
      ],
      "reference_response": null,
      "complete_output": [
        "完整的回答"
      ]
    }
  ]
}
```

选手最终只需提交 **一个 JSON 文件**，其中每个 sample 的格式如上所示，并将文件命名为 **`参赛队伍名称_result.json`** 进行提交。

**提交前自查：**

1. 文件顶层为 **JSON 数组** `[...]`，包含全部 **24 个** concept（L1_1–L1_8、L2_1–L2_8、L3_1–L3_8）。  
2. 每个 concept 下 **5 条** valid 样本，与官方 `valid.json` 一一对应。  
3. `pred` 为 steer 后的回答；`orig_pred` 按样例保持为 `[]`。  
4. 从 `all_generation_results_valid.json` 导出时，对齐 `concept_name`、`concept_description` 等字段名。

若得分为 0，常见原因包括：JSON 结构不符合评测脚本、缺少 concept/样本、`pred` 为空，或生成流程未正常完成。建议先按上文任选一种基线跑通 pipeline，再在此基础上改进方法。

# 复赛

本部分说明 CCKS2026 **复赛**的数据范围、实验设定与提交要求。基线框架与初赛相同，仍由 [EasyEdit2](https://github.com/zjunlp/EasyEdit/blob/main/README_2.md) 提供。

## 1. 数据集介绍

复赛使用的评测数据与初赛**保持一致**：

| 项目 | 说明 |
|------|------|
| 数据集 | SteerEval **Personality** 子集 |
| 训练（估计向量） | `personality/train.json`（规则同初赛，**不得**使用 valid 训练） |
| 评测 | `personality/valid.json`（**validation split**） |

数据格式、下载方式与本地路径见 [初赛 · 2. 数据集介绍](#2-数据集介绍)。

## 2. 实验设定

- **评测 split：** 与初赛相同，在 **validation split** 上评估与排名。  
- **基座模型：** 仍为 **Qwen3-4B-Instruct-2507**，方法须为激活干预类 Steering（禁止仅用 Prompt）。  
- **与初赛的差异：** 复赛侧重**方法效果**与**可复现性**；参赛队伍需提交**完整可运行代码**，组委会将进行**人工复现验证**，确保结果真实、可重复。  
- **基线参考：** 初赛提供了 **RePS**、**CAA** 两套示例命令（均为 `layers=18`，`multipliers=3`），见 [初赛 · 4. 快速运行](#4-快速运行)；参数与 method 均可自行调整。

## 3. 提交与复现要求

📌 **复现要求：** 参赛队伍需提交完整的可运行代码（含环境说明、数据与模型路径配置、训练/生成脚本及依赖版本），组委会将进行人工复现验证。若无法复现提交结果，将按赛规处理。

**结果文件：**

- 提交格式与初赛相同，仍为 **`参赛队伍名称_result.json`**，单条 sample 结构见 [初赛 · 6. 文件提交](#6-文件提交)。  
- 须覆盖全部 concept 及对应 valid 问题，`pred` 为施加 Steering 后的模型回答。
